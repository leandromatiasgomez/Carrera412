Clonas el repositorio en tu computadora, lo abres y ves que está vacío. Decides abrir Visual Studio, seleccionas el repositorio, creas el archivo hola.txt y escribes “Mundo”. Después de eso, ejecutas el comando add y luego un commit. Pero recuerdas que tenías que escribir algo más, así que vuelves a abrir el Visual, escribes en el archivo “Hola Mundo.” y ejecutas un status para ver el resumen de tu espacio de trabajo. Por último, ejecutas un add, un pull y después el push.


El archivo hola.txt primero se creó con el contenido “Mundo”, luego fue agregado y confirmado con un commit. Después se modificó a “Hola Mundo.”, por lo que al ejecutar git status el archivo quedó en estado modified. Tras ejecutar git add, pasó a staging area, pero como no se hizo un segundo commit antes del push, el cambio “Hola Mundo.” no se subió al repositorio remoto. Esta en Staged y Staged area
